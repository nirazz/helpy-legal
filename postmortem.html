      Historical Post-Mortems
      PM-001
      Browse feed showing own requests
      P1 Resolved
      When
      Feb 2026
      Summary
      Users saw their own help requests in the Browse feed, potentially causing confusion and accidental self-matches.
      Root Cause
      Auth state loaded asynchronously. When fetchRequests fired, user?.id was still null, so the "exclude own requests" filter had no value to filter on.
      Fix
      Added authLoading guard in useEffect - wait for auth to fully resolve before any user-scoped fetch.
      Lesson
      Always wait for auth before any user-scoped query. Use an authLoading flag in every screen that fetches user-specific data.
      PM-002
      Profile updates silently failing
      P1 Resolved
      When
      Feb 2026
      Summary
      All profile updates (name, avatar, bio) appeared to succeed but changes were not saved to the database. Users had no indication anything was wrong.
      Root Cause
      The Supabase update payload included updated_at which is not a column on the profiles table. This caused the entire UPDATE to silently fail with no error thrown by the client.
      Fix
      Removed updated_at from all update calls. Added .select().single() after updates to detect RLS or column failures.
      Lesson
      Always use .select().single() after updates to catch silent failures. Never assume a Supabase update succeeded just because no exception was thrown.
