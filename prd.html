Helpy - Neighbors Helping Neighbors
v1.0.2 · March 2026

A community help-matching app that turns the cold, transactional nature of neighborhood classifieds into a warm, trust-based experience - one swipe at a time.

📱 iOS Live 🇮🇱 Hebrew + English ⚡ Real-time Chat 🗺️ Location-aware 🔒 Phone Verified 🌙 Dark Mode

Section 01

Executive Summary

North Star Metric: Matches created per week - the moment two neighbors connect is when Helpy delivers its core value.

Helpy is a community help-matching mobile app that connects neighbors who need help with neighbors who can provide it. Using a swipe-based interface, Helpy reduces the friction of asking for and offering help within a local radius - turning everyday needs like moving furniture, tech support, or grocery runs into opportunities for genuine community connection. The app is live on the iOS App Store as "Helpy - Neighborly Help," free to use, and designed for dense urban neighborhoods starting with Tel Aviv. Unlike WhatsApp groups (too noisy), Facebook Marketplace (too transactional), or TaskRabbit (requires payment), Helpy is built specifically for the informal, trust-first exchange that actually happens between neighbors.

Transactional, anonymous, optimized for commerce not favors
💳
TaskRabbit / Fiverr
Payment required - creates a barrier for simple neighborly favors
🤐
Not Asking at All

Why Now

Urban isolation is a growing social problem. Post-COVID, people are more geographically aware of their neighbors yet more socially disconnected. Mobile-first, location-aware apps have normalized the "swipe to match" pattern. The timing is right for a trust-based community help layer on top of where people already live.

Section 05

User Stories

US-01 Post a Help Request ▶
As a user who needs help, I want to post a request with title, category, urgency, and photos, so that nearby neighbors can discover and offer to help me.
Acceptance Criteria
  • Title & description min 10 characters; category & urgency required
  • Up to 3 photos can be attached
  • High urgency capped at 2/week; Medium at 5/week - quota shown to user
  • Request appears in Browse feed for nearby helpers within 10km
  • Push notification sent to nearby available users for High urgency
  • Request auto-expires after 7 days with no accepted match
US-02 Browse and Offer Help ▶
As a helper, I want to swipe through nearby requests filtered by my radius, so that I can quickly find and offer help for what I can fulfill.
Acceptance Criteria
  • Feed shows requests within configured radius (default 5km, max 10km)
  • Excludes: own requests, blocked users, reported users
  • Sorted by urgency tier first, then distance within tier
  • Tapping "Offer Help" requires phone verification
  • Celebratory bottom sheet confirms offer sent; Undo available for 4 seconds
  • Map view toggle shows all requests as pins; tap pin to inspect
US-03 Manage Incoming Offers ▶
As a requester, I want to see who has offered to help and accept or decline them, so that I can choose the right helper.
Acceptance Criteria
  • MyRequests tab shows Active and History tabs with pending offer count
  • Each offer shows helper name, avatar, rating, and verification badge
  • Accepting an offer creates a match and opens chat immediately
  • Only one offer can be accepted per request
  • Declining removes offer silently (helper not notified)
US-04 Real-Time Chat ▶
As a matched user, I want to chat in real time with my match, so that we can coordinate the details of the exchange.
Acceptance Criteria
  • Chat opens automatically after offer is accepted
  • Optimistic send: message appears instantly before server confirms
  • Quick reply suggestions shown above input bar
  • Pending helpers see status card; cannot message until accepted
  • Chat persists in History after request is closed
US-05 Complete & Rate the Helper ▶
As a requester, I want to mark my request complete and rate the helper, so that helpers build trusted reputations.
Acceptance Criteria
  • RatingModal opens on "Mark Complete" - 1-5 stars + optional comment
  • Rating immediately updates helper's average_rating and rating_count
  • Match moved to History; helper streak counter increments
  • Review wall on helper's public profile updated instantly
  • Rating cannot be edited after submission
US-06 Phone Verification Gate ▶
As a new user, I want to verify my phone before posting or helping, so that the community knows I'm a real, accountable person.
Acceptance Criteria
  • Required before: posting a request OR tapping "Offer Help"
  • Two-step flow: Israeli phone → SMS OTP via Twilio
  • Auto E.164 formatting (0528... → +97252...)
  • phone_verified = true saved to profile; shown once ever
US-07 View Public Profile & Reviews ▶
As a requester considering an offer, I want to view the helper's public profile and ratings, so that I can make an informed trust decision.
Acceptance Criteria
  • Tappable avatar/name on Browse cards, offers, and Chat
  • Shows: name, avatar, verification badge, rating, help count, review wall
  • No PII exposed (no email, phone, exact coordinates)
US-08 Report and Block a User ▶
As a user encountering bad behavior, I want to report or block someone, so that I feel safe and the community stays trustworthy.
Acceptance Criteria
  • "..." menu on Browse cards and Chat header
  • Block auto-fires alongside report submission
  • Blocked users immediately disappear from feed (bidirectional)
  • Blocked users list on ProfileScreen with Unblock option
  • Reports reviewed by support; 24hr response SLA communicated
US-09 Full Hebrew / RTL Experience ▶
As a Hebrew-speaking user, I want the app to display fully in Hebrew with RTL layout, so that I can use it comfortably in my native language.
Acceptance Criteria
  • All UI text, alerts, and navigation have Hebrew translations (200+ keys)
  • RTL layout applied globally when Hebrew is selected
  • Distance shown in km in Hebrew mode, miles in English
  • Language preference persisted to Supabase profile
US-10 Account Deletion ▶
As a user leaving the platform, I want to permanently delete my account and all data, so that my information is not retained.
Acceptance Criteria
  • Two-step confirmation alert before deletion
  • Cascades: profile, requests, matches, messages, ratings, reports
  • User signed out immediately; auth account deleted from Supabase
  • Irreversible; no grace period (MVP)

Section 07

Non-Functional Requirements

⚡Performance
  • Browse feed load: <2s on 4G
  • Chat send (optimistic): instant; server <500ms
  • Image upload (3 photos): <5s
  • Auth timeout: 15s max (Google Sign-In)
  • Cold start to usable screen: <3s
🔒Security (CISO Baseline)
  • RLS on all 7 DB tables; no unauth data access
  • Service role key only in Edge Functions (server-side)
  • Phone verification enforced server-side via Twilio
  • Content filter on all UGC before insert
  • No PII in public API responses
  • Account deletion cascades all user data
🔐Privacy
  • Phone not exposed in public profile
  • Distance shown rounded (not exact coords)
  • Push tokens cleared on sign-out
  • GDPR-aware: EU Frankfurt data center
  • User can delete all data at any time
🏗️Reliability
  • RPC failures fall back to direct table queries
  • Auth state persisted via AsyncStorage
  • 5-15s timeouts on all critical async ops
  • Error boundaries with graceful fallback UI
  • Sentry crash reporting in production
♿Accessibility
  • Min touch target: 44x44pt (Apple HIG)
  • accessibilityLabel on all interactive elements
  • Full RTL layout support
  • WCAG AA color contrast (light + dark mode)

Section 08

User Flow (Happy Path)

1
Download & Onboard
User downloads app → 3 onboarding slides (skippable) → Auth screen
2
Sign In
Tap "Continue with Google" (or Apple) → OAuth completes → TermsModal on first sign-in → Auto-profile created → Location fetched → Lands on Browse tab
3
Phone Verification
First time tapping "Offer Help" or Post tab → PhoneVerifyModal → Enter number → SMS OTP → Verified once, never shown again
4
Post a Request (Requester)
Post tab → Title, description, category, urgency, 0-3 photos → Submit → Inserted to DB → Redirected to MyRequests → Push sent to nearby helpers if High urgency
5
Browse & Offer Help (Helper)
Browse tab → Swipe cards or use map view → Tap "Offer Help" → Celebration sheet ("Offer Sent!") + 4s Undo → match record created (status: pending)
6
Accept an Offer (Requester)
Offer badge on MyRequests → Tap request → View helpers → Tap heart (accept) → match status = accepted → Push to helper → Chat opens automatically
7
Coordinate via Chat
Both users message in real time → Arrange logistics and details → Helper shows up and helps
8
Complete & Rate
Requester taps "Mark Complete" → RatingModal (1-5 stars + comment) → Rating saved → average_rating updated → Request moves to History → Helper streak increments
✨
Build Reputation
Helper's public profile shows updated rating + completed count → Future requesters see trust signals → Virtuous cycle begins

Section 10

Risks & Mitigations

📭Cold Start / Empty Feed
Likelihood: HighImpact: High
New users open Browse to an empty feed. Zero value delivered, user churns immediately. Especially acute at 16 users.
MitigationSeed feed with sample requests at launch. Social proof pill shows activity. First-time nudge on empty state. Invite feature drives supply + demand together.
⚠️Trust & Safety Incident
Likelihood: MediumImpact: Critical
Bad actor uses platform to scam, harass, or harm a neighbor. One incident at this scale could end the app.
MitigationPhone verification (real identity). Report & Block live. ToS with zero-tolerance. Age gate 18+. Content filter. 24hr support response SLA. Bidirectional blocking.
📉Low Match Rate
Likelihood: Med-HighImpact: High
Requests posted but no helper offers within radius. Requesters lose faith and stop posting - breaking the core loop.
MitigationUrgency push to nearby helpers. Re-engagement after 12h no-offer. Feed priority sort for high-urgency. Auto-expiry prevents stale feed.
🔌Infrastructure Failure
Likelihood: LowImpact: Medium
Supabase outage causes app to hang. Users associate reliability issues with the product.
MitigationAll network calls have 5-15s timeouts. RPC falls back to table queries. Offline detection banner. Sentry captures prod errors. AsyncStorage caches auth.
🍎App Store Policy Change
Likelihood: MediumImpact: Medium
Apple / Google policy change requires significant rework. Already rejected 5x by Apple before approval.
MitigationUGC safety already robust. Privacy nutrition labels current. ToS legally reviewed (19 sections). Subscribe to policy newsletters.
